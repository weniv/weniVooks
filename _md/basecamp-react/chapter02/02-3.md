# 1. 페이지 구성요소의 컴포넌트화

- React의 핵심은 `컴포넌트`에 있습니다. 리액트에서 컴포넌트는 단순한 코드 조각이 아니라, 애플리케이션의 근간이자 핵심입니다. 컴포넌트를 통해 우리는 복잡한 UI를 이해하기 쉽고, 관리하기 쉬운 조각으로 나눌 수 있습니다. 이는 마치 레고 블록과 같아서, 각 컴포넌트를 조합하여 거대하고 복잡한 시스템을 구축할 수 있습니다. 효과적인 컴포넌트 설계는 React 애플리케이션의 성공을 좌우하는 핵심 요소입니다.
- 즉, 리액트 라이브러리의 목적은 `컴포넌트를 사용자 정의 요소로 표현하는 것` 이라고 할 수 있으며, 웹 페이지를 작은 독립적인 부분으로 나누어 개발하고 관리하는 것을 말합니다.

## 1. 001.html

- React 개발에 있어 또 하나의 중요한 도구인 `바벨(babel)`을 소개하겠습니다. 바벨은 최신 문법을 무버전의 JS로 만들어서 모두가 호환 가능하게 만들어줍니다. JSX는 바닐라 JS가 아니므로 바벨 프리셋 리액트로 호환을 하여 코드를 변환해주는 과정을 거쳐야 하기 때문입니다.

# 2. React를 사용하는 이유

- React를 사용하는 주된 이유는 페이지 구성 요소를 컴포넌트화 하기 위함입니다. 이는 대규모 프로젝트에서 복잡도를 줄여 주기 때문에 보다 효율적인 관리 및 유지보수가 가능하게 합니다. 또한, 일정 규모 이상에서 속도를 향상시킬 수 있는 장점이 있습니다. 즉, 하나하나의 코드 조각들은 컴포넌트화 시키면 똑같은 패턴이 반복되므로 조립식으로 재사용 및 재활용이 용이합니다.
- 반면, 일정 규모가 되지 않는 간단한 싱글 페이지나 다수의 정적 페이지에서는 React 도입이 적절하지 않을 수 있습니다. 그러므로 프로젝트의 크기와 앞으로의 확장성을 고려하여 React를 도입할지, Redux를 도입할지의 적절한 선택을 하는 것이 필요합니다.

# 5. 실용 예제 고도화 작업

:::div{.callout}
지금까지 React의 컴포넌트가 어떻게 활용이 되는지 알아보았는데요. 😁 이번에는 컴포넌트 작업을 고도화 하는 시간을 가져보도록 하겠습니다!

:::

## 1. 전체 소스코드

[쇼핑몰만들기.zip](/images/chapter02-3/쇼핑몰만들기.zip)

- ul 태그에 children이 array로 들어갑니다.
- unique 키 값을 가지고 있어야 합니다.
- 모듈화 : 각각의 컴포넌트들을 다른 파일로 만들어 주는 작업
- 컴포넌트 생성 시, 관례적으로 첫 문자는 대문자로 합니다.
- HTML에서 Node 환경을 사용하지 않고 .html 파일과 .js파일만으로 노드 환경에서 React를 사용해 보는 것처럼 실습 가능

## 2. 주목할 점

- 컴포넌트의 재사용성
- `props` 의 구조분해할당
- `className`, `onClick` 등의 카멜 표기법
- 중괄호 하나(`{}`)는 값(`{console.log(100+100)}`, `{변수명}`, `{함수명()}`)
- 중괄호 2개는 style(javascript안에 object)

## 3. 앞으로 배울 중요한 내용

- 프로젝트 규모가 커지면 props로 전달하는 것이 많아져서 Redux를 사용합니다.
- 마치 백엔드 프레임워크처럼 URL 라우팅을 할 수 있어요. 리플레쉬 되는 것도 아니기 때문에 깜빡임도 없습니다!
- 컴포넌트는 매번 직접 만들어 사용해야 하나? No! 
npm으로 설치할 수 있는 컴포넌트도 있고, 공개되어 있는 디자인도 많습니다.(라이센스는 확인하셔야 합니다.)
    - ::a[class101의 컴포넌트(디자인)]{class='btn-link' href="https://www.figma.com/community/file/995265899930701059" target="_blank"}
    - ::a[컴포넌트 백과사전]{class='btn-link' href="https://storybook.js.org/showcase" target="_blank"}
    - ant.design
    - emotion.sh
- `onClick`, `onChange`에 들어가는 콜백함수(핸들러 함수)는 handle 접두사를 붙이는 것이 관례입니다. 예시) `handleInfoSubmit`
- props로 어떤 값을 전달하는 핸들러의 경우 on 접두사를 붙이는 것이 관례입니다. 예시)`onInfoSubmit`
