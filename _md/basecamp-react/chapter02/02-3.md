::img{width="300" alt="JSX Logo" src="/images/basecamp-react/chapter02-3/chapter02-3-1.png"}

JSX란 `JavaScript XML`의 약자로, React에서 사용되는 JavaScript의 확장 문법입니다. `작스`라고 읽으며, JS 코드 내에서 `HTML과 유사한 구문`을 사용하여 `마크업 구조`를 쉽게 작성할 수 있게 해주는 강력한 도구입니다. JSX는 컴포넌트 기반 개발을 용이하게 하여 `코드의 재사용성과 가독성`을 높이는 역할도 수행합니다. 이러한 JSX를 통해 개발자는 더 직관적이고 효율적인 React 애플리케이션을 구축할 수 있습니다.

:::div{.callout}
**마크업 구조**란 'h1, p, div' 등의 태그<>들을 사용해 페이지의 구조를 만드는 것을 말합니다.

:::

```jsx
const element = <h1>Hello, JSX!</h1>;
```

이 코드는 HTML처럼 보이지만, 실제로는 JS 객체를 생성하는 JSX 표현식입니다.

:::div{.callout}
.js파일을 `.jsx`로 확장자를 변경하면 `태그 자동완성`이 활성화 됩니다.

:::

# 1. JSX 특징

:::div{.callout}
my-app 폴더에 있는 node_modules 폴더는 꽤나 큰 용량을 차지하기 때문에 챕터가 변경될 때마다 삭제를 반복하면서 진행하겠습니다.

:::

1. 컴포넌트 구조

우선, src 폴더에 있는 App.jsx 파일을 수정하면서 시작합니다.

`App.jsx`

```jsx
function App() {
  return (
    <div className="App">
      <h1>hello world</h1>
    </div>
  );
}

export default App;
```

HTML과 JS에 익숙한 분들에게는 위와 같은 React의 return() 구문이 낯설 수 있습니다. 하지만 이러한 구조는 컴포넌트를 효과적으로 만들고 관리하는 데 큰 도움이 됩니다. 이 방식을 통해 UI 요소를 더 명확하게 정의하고, 컴포넌트의 재사용성을 높일 수 있습니다.

이제, App 함수를 한 줄로 실행을 해보도록 하겠습니다. 아래 코드로 수정 후, 터미널에서 npm start로 실행해 보시길 바랍니다.

```jsx
function App() {
  return <h1>hello world</h1> {/* 한 줄 실행 시, 괄호() 생략 가능 */}
}

export default App;
```

2. `단일 부모 요소`

컴포넌트의 최상위 부모 요소는 오직 1개 입니다. 2개의 <h1> 태그를 사용하여 살펴 보겠습니다.

```jsx
function App() {
  return (
    <h1>hello world</h1>
    <h1>hello world</h1>
  );
}
```

**그런데,** 위 코드를 입력하였더니 컴파일에 실패했습니다. 그 이유는 무엇일까요?

::img{width="400" alt="Compiled failed" src="/images/basecamp-react/chapter02-3/chapter02-3-2.png"}

JSX로 표현할 때에는 하나의 부모만 존재할 수 있습니다. 즉, 최상위에는 한 개의 태그만 있어야 한다는 것입니다. 아래와 같이 코드를 수정하면 실행이 잘 되는 것을 확인할 수 있습니다.

```jsx
function App() {
  return (
    // div 태그는 최상위 수단으로 이용
    <div>
      <h1>hello world</h1>
      <h1>hello world</h1>
    </div>
  );
}
```

3. `주석`

JSX 내에서의 주석은 `{/**/}`로, 중괄호{}로 감싸준 형태를 사용합니다.

```jsx
function App() {
  // 여기는 JS 주석 표시와 동일합니다.
  console.log('주석입니다.');
  return (
    <div>
      {/* 최상위태그는 하나만! */}
      {/* 여기서부터는 JSX 문법 주석을 사용해야 합니다.*/}
      <h1>hello world</h1>
      <h1>hello world</h1>
      {/* 태그는 무조건 닫아줘야 합니다! 그렇지 않으면 에러가 발생합니다. */}
      <div />
      {/* 이런식으로 바로 닫히게 할 수도 있습니다. */}
    </div>
  );
}
```

# 2. 클래스와 스타일

이번에는 태그에 `class`를 넣고 `style`을 적용해 보겠습니다.

스타일을 정의할 때, JSX에서는 class가 아닌 `className`을 사용합니다. 여기에서 HTML과 차이점이 드러나게 됩니다. 바로 `-`를 사용할 수 없다는 사실입니다. 이러한 이유로 JSX는 단어와 단어를 대문자로 구분하여 작성합니다. 'background-color' 대신 `backgroundColor`라고 표현하는 것을 예시로 들 수 있습니다.

:::div{.callout}
통상적으로, **컴포넌트명**은 `대문자`로 시작하고 **css파일명**은 `소문자`로 시작하게 합니다.

:::

`App.jsx`

```javascript
import './app.css';

function App() {
  return (
    <div>
      <h1>Hello World!</h1>
      <h2>Hello World!</h2>
      <p className="content"/>Hello World!</p>
    </div>
  );
}

export default App;
```

JSX에서 또 다른 주의하실 점은, in-line style을 정의할 때에 아래 코드와 같이 `오브젝트 형태`로 작성해야 한다는 것입니다.

```javascript
// 오브젝트 형태 예시
{
  "key":value,
}
```

이러한 오브젝트 형태로 짜여진 스타일을 아래 코드와 같이 적용해 볼까요?

`app.css`

```css
.content {
  color: red;
}
```

# 3. Babel

::img{width="300" alt="Babel Logo" src="/images/basecamp-react/chapter02-3/chapter02-3-5.png"}

React 개발에 있어 또 하나의 중요한 도구인 `바벨(Babel)`을 소개하겠습니다. 바벨은 `최신 JS 문법을 이전의 JS로 변환해주는 도구`입니다. 쉽게 말해서 `코드 번역기`라고 생각하면 됩니다.

Component는 주로 JSX라는 특별한 문법으로 작성됩니다. 하지만 브라우저는 이 JSX를 직접 이해하지 못합니다. 여기서 Babel이 필요합니다. Babel은 JSX를 일반 JavaScript로 변환하여 브라우저가 이해할 수 있게 만듭니다. 또한 Babel은 React에서 사용되는 최신 JavaScript 문법을 구형 브라우저에서도 동작할 수 있는 코드로 변환합니다. 이렇게 Babel은 React Component가 모든 환경에서 올바르게 동작할 수 있도록 도와주는 중요한 도구입니다.

개발자가 React로 웹사이트를 만들 때, 편리한 최신 JS와 JSX라는 특별한 문법을 사용하지만, 모든 웹 브라우저가 이 새로운 언어를 이해하는 것은 아닙니다. 이때, 바벨은 최신 문법으로 작성된 코드를 오래된 어떠한 브라우저들도 이해할 수 있는 이전 버전의 JS로 바꿔줍니다. 이렇게 바벨은 개발자가 편하게 코드를 쓸 수 있게 해주면서, 동시에 모든 사용자가 웹사이트를 잘 볼 수 있도록 도와주는 역할을 합니다.

`App.jsx`

```jsx
function App() {
  const name = '라이캣!';
  return (
    <div className="App">
      <h1>안녕, {name} 1호</h1>
    </div>
  );
}

export default App;
```

자바스크립트에서 변수를 만들고 이 변수를 활용 해보도록 하겠습니다. 기존 return() 위에 `const name = '라이캣!'` 으로 변수 하나를 생성하고, return() 안의 jsx 태그에 라이캣 대신 `중괄호`를 입력하여 **name**으로 변경해 보겠습니다.

중괄호{} 사용은 JS에 다양한 문법을 사용할 수 있게 합니다. 아래의 코드들을 한 줄씩 입력하면서 테스트 해보시길 바랍니다.

```jsx
{
  100 + 1;
}
{
  'hello' + 'world';
}
{
  [1, 2, 3].map((x) => x ** 2);
}
{
  함수();
}
{
  변수;
}
{
  값 ? 'one' : 'two';
}
```

# 4. 실습

::img{width="400" alt="New UI" src="/images/basecamp-react/chapter02-3/chapter02-3-6.png"}

지금까지 배운 내용을 바탕으로 위 사진과 같은 UI를 구현해 보시길 바랍니다. 조건은 다음과 같습니다.

1. 시간은 직접 입력이 아닌, Data 객체값을 출력하게 합니다.
2. 새로고침할 때마다 현재 시간을 표시합니다. (JS에서 제공하는 Date 활용)
3. 개성있는 스타일을 적용해 보세요.

완성 예시 코드는 다음과 같습니다.

```jsx
import './app.css';

function App() {
  const name = '라이캣!';
  const someStyle = { backgroundColor: 'black', color: 'white' };
  const today = new Date();
  const year = today.getFullYear();
  const month = today.getMonth() + 1;
  const date = today.getDate();
  const hour = today.getHours();
  const min = today.getMinutes();
  const sec = today.getSeconds();

  return (
    <div>
      <div>
        <h1 style={someStyle}>안녕, {name} 1호</h1>
        <h1>안녕, 라이캣 2호!</h1>
        <div className="newClass" />
      </div>
      <div style={{ backgroundColor: 'black', color: 'white' }}>
        <h1 style={{ color: 'red' }}>년 : {year}</h1>
        <h1>
          월/일 : {month}/{date}
        </h1>
        <h1>
          시간 : {hour}시 {min}분 {sec}초
        </h1>
      </div>
    </div>
  );
}

export default App;
```
