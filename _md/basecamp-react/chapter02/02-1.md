# 1.1. Component 개념

`컴포넌트`는 React의 핵심 구성 요소입니다. 단순한 코드 조각 이상으로, 각각의 컴포넌트를 마치 레고 블록처럼, 조합하여 거대하고 복잡한 시스템을 구축할 수 있습니다. 이를 통해 복잡한 UI를 보다 효율적으로 이해하고 관리할 수 있게 해줍니다. 효과적인 컴포넌트 설계는 React 애플리케이션의 성공을 좌우한다고 해도 과언이 아닌 매우 중요한 과정입니다.

React의 핵심 개념인 Component는 `웹 페이지의 구성 요소를 사용자 정의 요소로 표현하는 방식`을 의미합니다. React 라이브러리의 주요 목적은 이를 통해 웹 애플리케이션을 작고 독립적인 부분들로 나누어 개발하고 관리하는 것입니다. 이러한 접근 방식은 복잡한 UI(사용자 인터페이스)를 보다 효율적으로 구축하고 유지보수할 수 있게 합니다.

# 1.2. Babel이란?

:::figure
::img{width="200" alt="Babel Logo" src="/images/basecamp-react/chapter02-1/chapter02-1-1.png"}
::figcaption[출처:https://medium.com/@larkigdrs/babel-babel-polyfill-%EC%99%9C-%EA%B0%99%EC%9D%B4-%EC%82%AC%EC%9A%A9%ED%95%A0%EA%B9%8C-b5f806ae64a1]
:::

React 개발에 있어 또 하나의 중요한 도구인 `바벨(Babel)`을 소개하겠습니다. 바벨은 `최신 JS 문법을 이전의 JS로 변환해주는 도구`입니다. 쉽게 말해서 `코드 번역기`라고 생각하면 됩니다. 개발자가 React로 웹사이트를 만들 때, 편리한 최신 JS와 JSX라는 특별한 문법을 사용하지만, 모든 웹 브라우저가 이 새로운 언어를 이해하는 것은 아닙니다. 이때, 바벨은 최신 문법으로 작성된 코드를 오래된 어떠한 브라우저들도 이해할 수 있는 이전 버전의 JS로 바꿔줍니다. 이렇게 바벨은 개발자가 편하게 코드를 쓸 수 있게 해주면서, 동시에 모든 사용자가 웹사이트를 잘 볼 수 있도록 도와주는 역할을 합니다.

:::div{.callout}
이러한 Component와 Babel의 이해를 바탕으로, 이번 장에서는 React의 다양한 기능 중 프로젝트에 `필요한 부분만을 선택하여 활용`하는 실습을 해보도록 하겠습니다.

:::

# 1.3. 실습

## 1.3.1. 1단계

`기본 코드`

```html
<!DOCTYPE html>
<html lang="ko">
    <head>
        <meta charset="UTF-8" />
        <meta http-equiv="X-UA-Compatible" content="IE=edge" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title></title>
    </head>
    <body></body>
</html>
```

:::div{.callout}
아래의 코드는 `<main/>` 등의 태그가 JSX 문법으로 사용되었으며, JSX를 사용하지 않고도 `Template Literal` 문법과 `InnerHTML`만을 사용하여 표현할 수 있다는 것을 보여 줍니다.

:::

`001/001.html`

```html
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
    </head>
    <body>
        <div>
        header
        <div>menu</div>
        </div>
        <div>
        main
        <h1>hello worlrd 002.html</h1>
        <p>hello world 002.html</p>
        </div>
        <div>product div*20(best seller 옷)</div>
        <div>footer</div>
        <div id="root"></div>
        <script src="https://unpkg.com/react@17/umd/react development.js" crossorigin></script>
        <script src="https://unpkg.com/react-dom@17/umd react-dom.development.js" crossorigin></script>
        <script src="https://cdnjs.cloudflare.com/ajax/1ibs/babel-standalone/6.26.0/babel.min.js" integrity="sha512-kp7YHLxuJDJcOzStgd6vtpxr4ZU9kjn77e6dBsivSz+pUuAuMlE2UTdKB7jjsWT84qbS8kdCWHPETnP/ctrFsA==" crossorigin="anonymous" referrerpolicy="no-referrer"></script>
        <script type="text/babel">
            const header = (
                <div>
                    <h1>header</h1>
                    <div>menu</div>
                </div>
            );
            const main = (
                <div>
                    <h1>main</h1>
                    <h3>hello world</h3>
                    <p>hello world</p>
                </div>
            );
            const product = (
                <div>
                    <h1>product</h1>
                    div{best seller 옷}*20
                </div>
            );
            const footer = (
                <div>
                    <h1>footer</h1>
                </div>
            );
            const body = (
                <div>
                    <h1>body h1이다!</h1>
                    {header}
                    {main}
                    {product}
                    {footer}
                </div>
            );
            const domContainer = document.querySelector('#root');
            ReactDOM.render(body, domContainer);
            console.dir(ReactDOM);
        </script>
    </body>
</html>
```

아래 코드에서 `소괄호()`는 Template Literal처럼 개행을 묶어 주는 역할을 하며, `최상위 태그`는 유일한 존재로서 하위 태그들을 감싸안아 묶어주고 있는 것을 확인할 수 있습니다.

:::
