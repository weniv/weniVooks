여기서 잠시, React와 Redux에서 사용되는 `상태 관리 방식`에 대해 간단히 알아보겠습니다. React 개발에서 중요한 과제 중 하나는 효율적인 `데이터 관리`입니다. 여러 컴포넌트를 거쳐 데이터를 전달하는 과정은 번거롭고 비효율적일 수 있는데, 이러한 문제를 해결하기 위해 `전역 상태 관리` 개념이 등장합니다. React는 `Context API`를 통해 props만으로 여러 단계의 컴포넌트를 거치지 않고 필요한 곳에 직접 데이터를 전달할 수 있습니다. 더 나아가, 4장에서 배울 Redux는 `데이터`를 통해 어떤 컴포넌트에서든 직접 접근하여 관리 및 사용이 가능합니다.

# 2. 고도화 작업

지금까지 쇼핑몰 페이지를 구현해 보면서, React의 컴포넌트가 어떻게 활용이 되는지 알아 보았습니다. 이번에는 이러한 컴포넌트 작업에 대한 고도화를 어떻게 하는지 알아보는 시간을 가져보겠습니다. 참고할 코드가 필요하시다면 아래 zip파일을 다운 받아 주세요.

[쇼핑몰만들기.zip](/images/chapter02-9/쇼핑몰만들기.zip)

:::div{.box}
우선, React의 효율적인 개발 환경을 구축하는 데에 도움이 되는 접근 방식을 알아보겠습니다. 첫번째, React에서 각 컴포넌트는 `unique(고유한) key 값`을 가져야 합니다. 이는 컴포넌트를 효율적으로 관리하고 업데이트하는 데 필수적입니다. 두번째, 각 컴포넌트를 별도의 파일로 분리하는 `모듈화 방식`을 권장합니다. 이 접근법은 코드의 재사용성을 높이고, 유지보수를 용이하게 하며, 전체 프로젝트 구조를 더 체계적으로 만드는 데 도움이 됩니다. 세번째, Node.js 환경 설정 없이도 단순히 `HTML 파일과 JS 파일만으로도 React 기본적인 기능을 경험`해볼 수 있습니다.
:::

:::div{.callout}
앞으로의 학습 과정에서 다음과 같은 내용을 다룰 예정이오니, 가볍게 읽어주시길 바랍니다.

1. 대규모 프로젝트에서는 `Redux`와 같은 상태 관리 도구를 사용하여 props 전달의 복잡성을 해결합니다.
2. `라우팅`을 통해, SPA(Single Page Application)에서 페이지 전환을 매끄럽게 처리할 수 있습니다.
3. props로 전달되는 핸들러에는 'on' 접두사를 사용하고, 이벤트 핸들러 함수명에는 'handle' 접수사를 사용하는 `이벤트 핸들링`을 사용하여 React를 보다 효율적이고 체계적으로 개발합니다.
