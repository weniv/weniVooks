여기서 잠시, React와 Redux에서 사용되는 `상태 관리 방식`에 대해 간단히 알아보겠습니다. React 개발에서 중요한 과제 중 하나는 효율적인 `데이터 관리`입니다. 여러 컴포넌트를 거쳐 데이터를 전달하는 과정은 번거롭고 비효율적일 수 있는데, 이러한 문제를 해결하기 위해 `전역 상태 관리` 개념이 등장합니다. React는 `Context API`를 통해 props만으로 여러 단계의 컴포넌트를 거치지 않고 필요한 곳에 직접 데이터를 전달할 수 있습니다. 더 나아가, 4장에서 배울 Redux는 `데이터`를 통해 어떤 컴포넌트에서든 직접 접근하여 관리 및 사용이 가능합니다.

1. 상세페이지(모달)

상세페이지를 모달 형태로 구현해 보겠습니다. 사용자가 상품을 클릭하면 모달 창이 열리고, 해당 상품의 상세 정보를 볼 수 있도록 만들어야 합니다. 이 모달에는 상품 이밎, 가격, 상세 설명 등이 포함되고, 사용자가 옵션을 선택할 수 있는 기능도 구현해야 합니다.

2. 장바구니(주문페이지)

사용자가 상품을 장바구니에 추가하고 제거할 수 있어야 하며, 수량 조절 또한 원활하게 이루어져야 합니다. 장바구니 페이지에서는 선택한 상품들의 총 가격을 계산하여 표시하고, 주문 과정으로 넘어갈 수 있도록 구현해야 합니다.

3. 좋아요 버튼

상품을 좋아요(찜)하는 기능은 클라이언트 내에서만 기록되고 동작합니다. 사용자가 상품을 좋아요 표시하면 그 정보를 로컬 스토리지에 저장하여 페이지를 새로고침해도 유지되도록 만듭니다. 또한, 좋아요 표시한 상품들을 모아볼 수 있는 별도의 페이지를 만드는 것도 좋은 아이디어입니다.

4. API

API 작업에 있어서는 GET 요청 외의 POST, PUT, DELETE 요청은 실제로 서버의 데이터를 변경하지 않고 성공 메시지만 반환하도록 구현합니다. 이는 잘못된 데이터가 서버에 올라가는 것을 방지하기 위함입니다.

`GET`

```
* / - 메인화면, hello world 출력
* /mall - 전체 prodoct 출력
* /mall/:id - product 상세 정보
* /coupon - 전체 coupon 출력
* /coupon/:productid - coupon 상세 정보
```

`POST(생성)`

```
* /mall - 실제 생성 되지 않고, success만 body에 추가해서 다시 던져줌
```

`PUT(수정)`

```
* /mall/:id - 실제 수정 되지 않고, success만 body에 추가해서 다시 던져줌
```

`DELETE(삭제)`

```
* /mall/:id - 실제 삭제 되지 않고, success만 body에 추가해서 다시 던져줌
```

5. 쿠폰 추가

쿠폰 기능을 추가할 때는 discount(할인가)와 discountRate(할인율)을 고려해야 합니다. '할인가=원가\*할인율'이라는 계산을 수행하도록 합니다. 쿠폰을 적용했을 때 최종 가격이 어떻게 변하는지 사용자에게 명확히 보여주어야 합니다.

6. 존재하지 않는 상품 처리

존재하지 않는 상품에 대한 처리도 중요합니다. 사용자가 잘못된 URL로 접근했을 때 적절한 오류 페이지를 보여주고, 메인 페이지로 돌아갈 수 있는 링크를 제공해야 합니다.

7. 뒤로가기 및 앞으로가기

브라우저의 뒤로가기와 앞으로가기 기능이 올바르게 작동하도록 React Router를 활용하여 라우팅을 구현해야 합니다. 이를 통해 사용자가 자연스럽게 웹사이트를 탐색할 수 있게 됩니다.

8. 스타일 적용

마지막으로, styled-components를 사용하여 동적으로 스타일을 적용합니다. 이를 통해 컴포넌트별로 스타일을 관리하고, 필요에 따라 동적으로 스타일을 변경할 수 있습니다.

이 외에도 전역 상태 관리, 성능 최적화, 단위 테스트 작성, 웹 접근성 개선 등을 고려하면 더욱 완성도 높은 웹 애플리케이션을 만들 수 있을 것입니다. 이러한 요소들은 실제 프로덕션 환경에서 매우 중요하므로, 시간이 허락한다면 이 부분들도 구현해 보는 것이 좋겠습니다.
