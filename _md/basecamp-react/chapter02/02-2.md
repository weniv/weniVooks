# 1. 소개

:::div{.callout}
교안은 React 공식문서(첫번째 링크)를 참고하고 있으며, Stack Overflow에서 제공하는 조사(두번째 링크)를 통해 2023년 기준 React의 사용률을 파악할 수 있습니다.

::a[React - 사용자 인터페이스를 만들기 위한 JavaScript 라이브러리]{class='btn-link' href="https://ko.reactjs.org/" target="\_blank"}

::a[Stack Overflow Developer Survey 2023]{class='btn-link' href="https://survey.stackoverflow.co/2023/#section-admired-and-desired-web-frameworks-and-technologies" target="\_blank"}

:::

React는 `Meta(구 Facebook)에서 관리`하고 있는 오픈소스 프로젝트로, 민간에서 관리하는 것에 비해 기업 차원에서 안정적인 유지보수와 빠른 주기의 업데이트가 이루어지고 있습니다.

그리고, `컴포넌트 기반 아키텍처`를 가지고 있습니다. 이는 UI를 독립적이고 재사용 가능한 조각인 '컴포넌트'로 나누어 자체적인 로직 및 뷰를 가지는데, 이 접근 방식은 코드의 `재사용성`을 높이고 유지보수를 용이하게 합니다.

```javascript
// 컴포넌트 예시
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
```

또한, `가상(Virtual) DOM`이라는 실제 DOM의 가벼운 복사본을 사용합니다. 상태가 변경될 때 먼저 가상 DOM을 업데이트하고, 이전의 가상 DOM과 비교하여 실제 변경이 필요한 부분만 실제 DOM에 적용합니다. 이러한 과정을 재조정(Reconciliation)이라 하며, 이를 통해 성능을 크게 향상 시킬 수 있습니다. 하지만, 때에 따라서 DOM이 더 빠를 수 있으니 상황에 맞추어 사용해야 합니다.

:::figure
::img{width="200" alt="DOM" src="/images/basecamp-react/chapter02-2/chapter02-2-2.png"}
::figcaption[<실제 돔 형식을 설명하는 사진>]
:::

```javascript
  /**
  * React로 생성된 DOM 예시 (JSX 이전)
  * 여기서 리액트 팀이 라이브 돔을 어떻게 관리하고자 했는지 힌트를 얻을 수 있습니다.
  */

  function Greeting({ name }) {
    return createElement(
      'h1', // 첫번째 인자는 만들 요소의 종류
      { className: 'greeting' }, // 두 번째 인자는 요소의 프로퍼티
      'Hello ', // 세번째 인자 이후로는 자식요소.
      createElement('i', null, name),
      '. Welcome!'
    );
  }

  // 렌더링 했을때 실제로 화면에 보이는 구조
  <h1 class="greeting">
    Hello <i>재현</i>. Welcome!
  </h1>

```

뿐만 아니라, `JSX 문법`을 사용합니다. JSX는 JavaScript의 확장 문법으로, HTML과 유사한 구문을 사용하여 React 요소를 생성합니다. JavaScript 코드 안에 마크업을 직접 작성할 수 있어 컴포넌트의 구조를 직관적으로 표현할 수 있습니다. 즉, 깜빡임이 없어 사용자 경험이 향상됩니다.

```jsx
  // JSX 문법 예시
  const element = <h1>Hello, world!</h1>;
```

또 다른 특징 중 하나는 `단방향 데이터 흐름`입니다. 이는 데이터가 항상 상위 컴포넌트에서 하위 컴포넌트로 흐르는 '하향식' 구조를 말합니다. 이 원칙은 애플리케이션의 상태 관리를 예측 가능하게 하고 디버깅을 용이하게 합니다. 상태 변경이 필요할 때는 `setState()` 메서드를 사용하여 명시적으로 업데이트합니다. 이로 인해 데이터의 흐름과 변화를 추적하기 쉬워져 애플리케이션의 안정성과 성능이 향상됩니다.

```jsx
  // 단방향 데이터 흐름 예시
  class Counter extends React.Component {
    state = { count: 0 };

    increment = () => {
      this.setState({ count: this.state.count + 1 });
    }

    render() {
      return (
        <div>
          <p>Count: {this.state.count}</p>
          <button onClick={this.increment}>Increment</button>
        </div>
      );
    }
  }
```

```jsx
  <!DOCTYPE html>
  <html>
    <head>
      <meta charset="UTF-8" />

    </head>
    <body>
      <h1>내 html에 리액트 1분만에 추가해보기!</h1>
      <p>단 1분만에 설치없이 리액트를 추가할 수 있다구요?</p>
      <div id="root"></div>
  		<script src="https://unpkg.com/react@18/umd/react.development.js" crossorigin></script>    // React 문법을 사용할 수 있게 합니다.
  	  <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js" crossorigin></script>    // React 문법을 사용할 수 있게 합니다.
      <script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>    // babel에서 JSX 문법을 사용할 수 있게 합니다.
      <script type="text/babel">    // babel에서 JSX 문법을 사용할 수 있게 합니다.
          // react 컴포넌트 정의하기
          const el = React.createElement;

          class HelloButton extends React.Component {
            constructor(props) {
                super(props);

                this.state = { liked: false };
            }

            render() {
                if (this.state.liked) {
                    return 'Hello World!';
                }

                return (
                  <button onClick={() => this.setState({ liked: true })}>
                      Hello
                  </button>
              );
            }
          }

          // DOM에 리액트 컴포넌트 랜더링하기
  				const domContainer = document.querySelector('#root');
  
  				// ReactDOM.createRoot 함수는 React 18부터 도입된 함수로, React 애플리케이션에서 루트 노드를 만들고, 그 루트 노드를 기반으로 리액트 앱을 렌더링하는 데 사용됩니다.
  				const root = ReactDOM.createRoot(domContainer);
  				root.render(el(HelloButton));
      </script>
    </body>
  </html>

```
