## 1. CDN과 babel 사용

### `코드`

```jsx
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>리액트 1분만에 추가해보기!</title>
  </head>
  <body>
    <h1>내 html에 리액트 1분만에 추가해보기!</h1>
    <p>단 1분만에 설치없이 리액트를 추가할 수 있다구요?</p>
    <div id="root"></div>
		<script src="https://unpkg.com/react@18/umd/react.development.js" crossorigin></script>
	  <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js" crossorigin></script>
    <script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>
    <script type="text/babel">
        // react 컴포넌트 정의하기
        const el = React.createElement;

        class HelloButton extends React.Component {
          constructor(props) {
              super(props);

              this.state = { liked: false };
          }

          render() {
              if (this.state.liked) {
                  return 'Hello World!';
              }

              return (
                <button onClick={() => this.setState({ liked: true })}>
                    Hello
                </button>
            );
          }
        }

        // DOM에 리액트 컴포넌트 랜더링하기
				const domContainer = document.querySelector('#root');
				
				// ReactDOM.createRoot 함수는 React 18부터 도입된 함수로, React 애플리케이션에서 루트 노드를 만들고, 그 루트 노드를 기반으로 리액트 앱을 렌더링하는 데 사용됩니다.
				const root = ReactDOM.createRoot(domContainer);
				root.render(el(HelloButton));
    </script>
  </body>
</html>
```

### `설명`
- line 13, 14 : React 문법 사용할 수 있게 함
- line 15, 16 : babel에서 JSX 문법을 사용할 수 있게 함

# 2. React란?
- `Meta(구 Facebook)에서 관리`하고 있는 오픈소스 프로젝트이기 때문에 주기적인 업그레이드와 그 속도 또한 빠른 편입니다.

![chapter01-2-13.png](/images/basecamp-react/chapter01-2/chapter01-2-13.png)

*<[State of JavaScript](https://2022.stateofjs.com/en-US/libraries/front-end-frameworks/)에서 제공하는 위 그래프(사진)를 통해, 민간에서 관리하는 Vue.js에 비해 회사 차원에서 관리하는 React와 Angular의 점유율이 지속적으로 높은 것을 확인할 수 있습니다.>*

- `컴포넌트 기반 아키텍처`를 가지고 있습니다. 이는 UI를 독립적이고 재사용 가능한 조각인 '컴포넌트'로 나누어 자체적인 로직 및 뷰를 가지는데, 이 접근 방식은 코드의 `재사용성`을 높이고 `유지보수를 용이`하게 합니다. `'props(재료)' -> '컴포넌트(공장)' -> '요소(생산품)'` 순으로 생산이 진행됩니다.

```python
  # 예시
  function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
```

- `가상(Virtual) DOM`이라는 실제 DOM의 가벼운 복사본을 사용합니다. Object 형식으로 되어있으며 상태가 변경될 때 먼저 가상 DOM을 업데이트하고, 이전의 가상 DOM과 비교하여 실제 변경(바뀌는)이 필요한 부분만 실제 DOM에 적용합니다. 이러한 과정을 `재조정(Reconciliation)`이라고 하며, 이를 통해 성능을 크게 향상 시킬 수 있습니다. 하지만, 때에 따라서 DOM이 더 빠를 수 있으니 상황에 맞추어 사용해야 합니다.

![chapter01-2-14.png](/images/basecamp-react/chapter01-2/chapter01-2-14.png)

*<실제 돔 형식을 설명하는 사진>*

```javascript
  // 리액트로 만드는 돔. JSX가 나오기 이전에 리액트는 아래처럼 돔을 생성했습니다. 여기서 리액트 팀이 라이브 돔을 어떻게 관리하고자 했는지 힌트를 얻을 수 있습니다.
  function Greeting({ name }) {
    return createElement(
      'h1', // 첫번째 인자는 만들 요소의 종류
      { className: 'greeting' }, // 두 번째 인자는 요소의 프로퍼티
      'Hello ', // 세번째 인자 이후로는 자식요소.
      createElement('i', null, name),
      '. Welcome!'
    );
  }

  // 렌더링 했을때 실제로 화면에 보이는 구조
  <h1 class="greeting">
    Hello <i>재현</i>. Welcome!
  </h1>

```

- `JSX 문법`을 사용합니다. JSX는 JS에 XML이 붙여진 JavaScript의 확장 문법으로, HTML과 유사한 구문을 사용하여 React 엘리먼트를 생성합니다. JavaScript 코드 안에 마크업을 직접 작성할 수 있어 컴포넌트의 구조를 직관적으로 표현할 수 있습니다. 즉, 깜빡임이 없어 사용자 경험이 향상됩니다.

*<React 공식문서로 이동하는 웹페이지>*

```jsx
  // 예시
  const element = <h1>Hello, world!</h1>;
```

- `단방향 데이터 흐름`으로 진행됩니다. 항상 부모 컴포넌트에서 자식 컴포넌트로 흐릅니다. '하향식' 데이터 흐름이라고도 하며, 이 원칙은 애플리케이션의 상탱 관리를 예측 가능하고 디버깅하기 쉽게 만듭니다.

- `상태(State)`와 `속성(Props)`을 가집니다. 상태란 컴포넌트 내부에서 관리는 데이터를 말합니다. 시간이 지남에 따라 변할 수 있는 특징을 갖고 있으며, setState() 메서드를 통해 업데이트됩니다. 또한, 속성은 부모 컴포넌트로부터 자식 컴포넌트로 전달되는 읽기 전용 데이터입니다. 컴포넌트는 자신의 props를 변경할 수 없다는 것을 의미합니다.

```jsx
  // 예시
  class Counter extends React.Component {
    state = { count: 0 };

    increment = () => {
      this.setState({ count: this.state.count + 1 });
    }

    render() {
      return (
        <div>
          <p>Count: {this.state.count}</p>
          <button onClick={this.increment}>Increment</button>
        </div>
      );
    }
  }
```
