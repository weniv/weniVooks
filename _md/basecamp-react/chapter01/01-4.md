## 1. 개념

:::div{.box}
- `Node.js`는 Chorme V8 JavaScript 엔진에다 살을 붙여(file, path, os..) 빌드된 `JavaScript 런타임(실행 환경)`입니다.
    - firefox - spidermonkey
    - explorer - chakara(edge는 v8사용)
    - 엔진이 다르기 때문에 호환되는 태그들도 다른 것입니다!
- `JavaScript 실행기`로써, 시스템에서 JavaScript를 실행할 수 있게 해 줍니다.
- javascript, HTML, CSS, WASM 어디서 돌아가나요?
- 웹 개발자가 자바스크립트로 (백엔드까지) 다 개발하고 싶을 때 활용됩니다.
- Node로 서버를 만들 수 있지만, 서버만 만들 수 있는 것은 아닙니다. 즉, 서버 자체는 아닙니다. `(Node ≠ 서버)`
:::

## 2. 특징

:::div{.box}
- `Non-Blocking I/O` : 비동기 Input 및 Output으로, 하나의 작업이 완료될 때까지 기다리지 않고 바로 다음 작업을 진행하는 방식입니다. ex) 점원이 한 번의 주문을 받은 커피가 나올 때까지 기다리는 것이 아닌, 손님1,2,3 주문을 일단 모두 받음
- `Single Thread` : 프로그램이 한 번에 하나의 작업만 순차적으로 실행하는 방식입니다. 이때, 프로그램 중단을 방지하기 위해 Error 처리는 필수적으로 이뤄져야 합니다. ex) 카페에 주문을 받는 점원이 한명 (+ 주문을 받지 않을 때의 error 처리)

## 1. 명령어 실행

1. 터미널을 열고 아래 명령어를 실행하면 React로 이루어진 프로젝트를 생성할 수 있습니다.

:::div{.callout}
아래 명령어를 사용하기 위해서는 Node 14.0.0 이상 또는 npm 5.6 이상의 버전이 필요 합니다. npx는 npm의 도구이며 npm의 5.2.0 이상에서 지원합니다.

:::

```bash
npx create-react-app my-app

# my-app이라는 폴더에 React와 모듈들이 함께 설치됩니다.
```

```bash
cd my-app
npm start
```

3. 터미널에 다음과 같은 문구와 웹페이지가 나타났다면 성공하셨습니다!

## 1. npm

- npm은 노드의 패키지 매니저 역할을 합니다.

:::div{.callout}
💡 npm과 npx가 아예 다른 것은 아니지만, npm과 달리 npx는 프로젝트를 시작하기 위해 필요한 다양한 설치를 한 번의 명령어로 세팅까지 해결해 주는 명령어 입니다. 즉, `npm install create-react-app`으로도 설치는 가능하지만 세팅은 사용자가 따로 해주어야 하므로 npx를 사용하는 것이 보다 효율적입니다.

:::

    ```jsx
    1.2.3
    // major.minor.patch
    // 메이저가 업데이트 되면 기존 버전과 호완이 안될 확율이 매우 높습니다! 업데이트 주의해주세요.
    ```

## 5. npm 명령어

- npm init : 초기 설정
- npm init -y : 초기 설정 한 번에 yes
- npm install <패키지명> or npm i <패키지명> : 설치
- npm install <패키지명> or npm i <패키지명> -D : 개발 전용으로 설치
- npm i -g <패키지명> : 전역설치
- npm uninstall <패키지명> : 삭제
- npm uninstall -g <패키지명> : 전역삭제
- npm ls : 설치된 패키지 확인
- npm ls -g : 전역 설치된 패키지 확인
- npm i --save : 5버전 이후로는 디폴트(명시하지 않아도 됩니다.)
