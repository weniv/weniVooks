# 1. 변수란?

## 1.1 변수는 가리키는 것

변수는 아래와 같은 포스트잇과 같습니다. 담겨져 있는 실체는 변하지 않아요. 다만 화장실에 `화장실`이라는 포스트잇을 붙여넣거나 `토일렛`이라는 포스트잇을 붙여놓으면, 그 화장실은 `화장실`이나 `토일렛`으로 불릴 수 있는 것이죠. 무엇이라 불리던 그 실체가 화장실인 것은 변하지 않습니다.

![](/images/python/chapter03/01-1.png)

더러는 변수를 상자로 표현하곤 합니다. 이것은 python에서 잘못된 비유라고 볼 수 있어요. 왜냐하면 실제로 python에서 변수 안에 값이 담기는 것은 아니거든요. 변수는 값과 아래처럼 화살표 관계가 됩니다.

![](/images/python/chapter03/01-2.png)
<img src="images/python/chapter03/01-2.png" width="50%">

예를 들어 화장실에 포스트잇이 3개, `화장실`, `토일렛`, `뒷간`이 붙어있다고 한다면 아래와 같은 화살표가 됩니다.

![](/images/python/chapter03/01-3.png)

## 1.2 변수의 생성

Python에서 변수를 만드는 것은 매우 간단합니다. 변수 이름을 정하고, '=' 기호를 사용하여 변수에 값을 할당하면 됩니다.

```python
age = 30
name = "Jun"
```

이 예제에서 'age'라는 변수에 30이라는 값을, 'name'이라는 변수에 "Jun"이라는 값을 할당했습니다. 이는 아래와 같은 화살표가 그려지는 것입니다.

![](/images/python/chapter03/01-4.png)

## 1.3 변수가 가리키는 값

```python
x = 10
y = x
z = y
```

만약 위와 같이 변수를 선언하였다면 화살표는 어떻게 만들어질까요?

![](/images/python/chapter03/01-5.png)

혹시 여러분이 위와 같이 변수가 만들어졌다고 생각하신다면 오해를 하고 있는 것입니다. 만약 위와 같은 화살표였다면 x가 가리키고 있는 값이 변했을 때 y와 z도 값이 변해야겠죠.

```python-exec
x = 10
y = x
z = y
x = 20
print(y, z) # 결과 값은 10입니다.
```

따라서 위와 같은 그림은 아래와 같이 그려지는 것이 맞습니다.

```python
x = 10

```

![](/images/python/chapter03/01-6.png)

그리고 2번째 예제는 아래와 같이 그려지는 것이죠.

```python-exec
x = 10
y = x
z = y # x = 10

x = 20
print(y, z) # 결과 값은 10입니다.
```

# 2. 변수 이름의 규칙

## 2.1 변수 이름 짓기

변수의 이름을 정할 때는 몇 가지 규칙을 따라야 합니다.

1. 변수 이름은 알파벳(대문자와 소문자), 숫자, 언더스코어(\_)로 구성됩니다.

   :::div{.callout}
   물론 아래와 같은 변수명이 안되는 것은 아니지만 관습적으로 허락하지 않습니다.

   ```python
   π = 100
   π + π
   ```

   :::

1. 변수 이름은 숫자로 시작할 수 없습니다. 1ten / 100_test (x) \_100test
1. 파이썬의 키워드(예: if, else, while 등)는 변수 이름으로 사용할 수 없습니다.

   아래 코드로 파이썬의 키워드를 알아낼 수 있습니다. 다만 이런 키워드를 지금 다 외워야 하는 것은 아니고, 수업을 진행하면서 하나씩 자연스럽게 습득할 수 있게 됩니다. 또 반대로 수업이 다 종료된 후 모르는 키워드가 있지는 않은지 살펴볼 수도 있습니다.

   ```python-exec
   import keyword

   print(keyword.kwlist)
   ```

   <!-- ```python
   # 출력
   ['False', 'None', 'True', 'and', 'as', 'assert', 'async', 'await', 'break', 'class', 'continue', 'def', 'del', 'elif', 'else', 'except', 'finally', 'for', 'from', 'global', 'if', 'import', 'in', 'is', 'lambda', 'nonlocal', 'not', 'or', 'pass', 'raise', 'return', 'try', 'while', 'with', 'yield']
   ``` -->

1. 변수 이름은 대소문자를 구분합니다. (예: 'name', 'Name', 'NAME'은 서로 다른 변수입니다.) Name ≠ name
1. 관습

   1. 일반 변수는 첫 문자를 소문자로 쓰는 것이 보통입니다.
   2. 사용하지 않는 변수는 언더스코어(`_`)로 명명합니다. test → \_test
   3. 클래스는 첫 문자를 대문자로 사용합니다. class Test():
   4. 변수 이름은 스네이크 표기법을 주로 사용합니다.

   :::div{.callout}
   스네이크 표기법은 언더스코어로 띄어쓰기를 표기하는 방법입니다. 뱀 같다 하여 스네이크라 합니다. 예를 들어 아래와 같이 표기될 수 있습니다.

   ```python
   daily_user_count = 100
   ```

   카멜 표기법은 언더스코어 대신 중간 문자를 대문자로 하는 표기법입니다. 낙타 등 같다 하여 카멜이라 합니다.

   ```python
   dailyUserCount = 100
   ```

   파스칼 표기법은 모든 단어에 앞 문자를 대문자로 표기하는 것입니다. 클래스 등에서 많이 사용합니다.

   ```python
   DailyUserCount = 100
   ```

   :::

# 3. 변수의 타입

‘변수의 타입은 무엇인가?’, ‘이런 타입이 왜 있는가?’, 이것에 대해 위 포스트잇 비유의 연장으로 말씀드려보도록 하겠습니다. 변수의 타입은 ‘그 변수를 어떻게 관리하고 처리하는가?’와 연결됩니다.

양말에 x라는 포스트잇이 붙어있다면 x의 타입은 결국 양말의 타입, 옷입니다. 옷은 입는 것이죠. 옷끼리 더하거나, 빼거나, 곱하는 것에 그 목적이 있지 않습니다.

![](/images/python/chapter03/01-8.png)

아래와 같이 1이라는 숫자에 y라는 포스트잇이 붙어 있다면 y의 타입은 결국 1, 숫자입니다. 숫자는 연산하는데 목적이 있습니다. 이것을 입을 수는 없죠.

![](/images/python/chapter03/01-9.png)

이렇게 변수의 타입은 그것을 어떻게 활용할지 정하게 됩니다. 예를 들어, 문자열에 경우에 덧셈은 이어 붙이고, 숫자의 덧셈은 두 수를 더합니다. 실제로 컴퓨터 입장에서 취급하는 것은 둘 다 숫자인데 말이죠. 기억하세요, 컴퓨터 입장에서는 모두 0과 1로 이뤄진 숫자입니다. 타입이 있으므로 이게 이어 붙이는 것인지, 실제 더하는 것인지 알 수 있는 것이죠.

```python-exec
x = 10
print(type(x)) # 출력: <class 'int'>

x = "Jun"
print(type(x)) # 출력: <class 'str'>
```

Python은 변수의 타입을 자동으로 결정합니다. 즉, 변수에 값을 할당하면, Python은 그 값의 타입에 따라 변수의 타입을 결정합니다. 양말에 포스트잇을 붙일 때 `양말(옷)` 이렇게 포스트잇을 붙일 필요가 없다는 얘기이죠.

이 예제에서, 처음에 'x'에 10을 할당하면, 'x'의 타입은 정수(int)가 됩니다. 그런 다음 'x'에 "Jun"을 할당하면, 'x'의 타입은 문자열(str)이 됩니다.

이것이 얼마나 편한지 파이썬으로 프로그래밍을 처음 접하시는 분들은 체감하지 못합니다. 다른 언어들은 일일이 이 타입을 지정해주어야 합니다. 아래처럼요.

```c
// c언어 예제
int x = 10;
char y = 'Jun';
```

타입을 확인하는 것은 type함수를 사용하고, 이 타입을 어떻게 처리하고, 어떤 기능이 있는지는 dir을 통해 확인합니다. 이 2개는 세트로 기억을 해주세요. 초급자 뿐만 아니라 중급자에게도 유용하게 사용됩니다. 다른 분야로 넘어가면 다른 자료형을 만나게 되는데 거기서도 이 2개를 확인하면 타입이 어떤 속성을 가지고 있는지 알 수 있거든요. 예를 들어보겠습니다.

```python-exec
x = 10
print(type(x))
print(dir(x))
```

```python
# 출력
<class 'int'> # int라는 이름을 가지고 있군요!
['__abs__', # 매직 메서드: 절대값이 되는군요!
'__add__', # 매직 메서드: 더하기가 되는군요!
'__and__', # 매직 메서드: 논리 연산이 되는군요!
# ...생략...
'__mul__', # 매직 메서드: 곱하기가 되는군요!
# ...생략...
'real', # 메서드: 실수부를 출력하는 메서드
'to_bytes' # 메서드: 바이트로 변환하는 메서드
]
```

dir의 출력은 크게 매직 메서드 부분과 메서드 부분으로 나눌 수 있습니다. 이에 대한 개념은 class에서 다루게 됩니다. 매직 메서드는 그 타입이 할 수 있는 일들을 나타냅니다. 더하거나, 빼거나, 곱하거나, 나누는 일을 정수는 할 수 있습니다. 반면에 str은 `__abs__`가 없으니 절대값이 되지 않습니다. `__sub__`도 없어 뺄셈도 안됩니다.

# 4. 변수 값 변경

변수에 저장된 값을 변경하려면 새 값을 할당하면 됩니다.

```python-exec
x = 10
print(x)

x = 20
print(x)
```

이 예제에서, 처음에 'x'에 10을 할당하고, 그런 다음 20을 할당하여 'x'의 값을 변경했습니다.

# 5. 변수 삭제

변수를 삭제하려면 'del' 키워드를 사용합니다.

```python-exec
x = 10
print(x)

del x
print(x)
```

이 예제에서, 'x'를 삭제한 후 'x'를 출력하려고 하면, Python은 에러를 발생시킵니다. 이는 'x'라는 변수가 더 이상 존재하지 않기 때문입니다.

# 6. 변수의 여러가지 타입

변수에는 참 여러가지 타입이 있습니다. 여기서 변수가 어떠한 형태를 가지고 있는지만 확인하고, 뒤에 챕터에서 하나씩 정복해가는 방법을 사용하도록 하겠습니다.

보통 아래와 같은 그룹으로 자료형을 부릅니다. 하지만 여기서 묶여진 그룹은 이해를 돕기 위한 개념적인 것이지 실제 하위 개념(하위 클래스 또는 동일한 부모)는 아닙니다. 실제로는 int, float, complex, bool이 같은 부모로 묶여 있습니다.

![](/images/python/chapter03/01-10.png)

- 숫자형: 숫자를 다룹니다.
  - 정수형(int): 양수와 0, 음수를 저장할 수 있습니다. 1 5 100 0 -30 -45
  - 실수형(float): 소숫점을 표현할 수 있습니다. 1.5 5.6 100.23 0.0 -30.53245
  - 복소수형(complex): 실수부와 허수부로 나누어 수를 표현할 수 있습니다.
- 논리 자료형(bool) : 논리 자료형 True, False을 다룹니다.
- 시퀀스: 순서가 있는 자료형입니다.
  - 문자열 자료형(str): 문자열을 저장합니다.
  - 리스트 자료형(list): 다른 자료형을 담을 수 있는 변경 가능한 자료형입니다.
  - 튜플 자료형(tuple): 다른 자료형을 담을 수 있는 변경 불가능한 자료형입니다.
- 컬렉션 혹은 컨테이너 자료형: 다른 자료형을 담을 수 있는 자료형입니다.
  - 리스트 자료형(list): 다른 자료형을 담을 수 있는 변경 가능한 자료형입니다.
  - 튜플 자료형(tuple): 다른 자료형을 담을 수 있는 변경 불가능한 자료형입니다.
  - 사전 자료형 또는 딕셔너리 자료형(dict): key와 value의 쌍으로 묶여 있는 key의 중복을 허락하지 않는 자료형입니다.
  - 집합 자료형 또는 셋 자료형(set): 값에 중복을 허락하지 않는 자료형입니다.
- None 자료형: None, 아무것도 없음을 명시하는 자료형입니다.

각 자료형이 어떤 형태로 되어 있는지 살펴보도록 하겠습니다. 여기서는 각각에 형태만 확인하고 가도록 하겠습니다.

아래 기술된 자료형은 Python에서 기본적으로 제공하고 있는 자료형입니다. 때로는 이러한 자료형이 아닌 외부에서 구현된 자료형을 공부해야 경우도 생깁니다.

```python
a = 10      # int, 정수형
b = -1      # int, 정수형
c = 0b110   # int, 정수형(2진법)
d = 0o56    # int, 정수형(8진법)
e = 0xAC    # int, 정수형(16진법)

f = 10.1    # float, 실수형

g = 10 + 2j # complex, 복소수

h = 'good'  # str, 문자열
i = '10'    # str, 문자열

j = True    # bool, 논리 자료형(부울형)

k = None    # NoneType

l = [1, 2, 3, 4]       # list, 리스트 자료형
m = ['one', 'two']     # list, 리스트 자료형
n = (1, 2, 3, 4)       # tuple, 튜플 자료형
o = ('one', 'two')     # tuple, 튜플 자료형
p = {'one':1, 'two':2} # dict, 딕셔너리 자료형
q = {1, 2, 3, 4}       # set, 셋 자료형

def r():    # function, 함수
    return

s = print # bulit-in function
t = lambda x:x**2 # function
u = int # type

class Character: # class, 클래스
    pass

hero = Character()    # instance
Villain = Character() # instance
```

이것을 출력할 때에는 3개를 세트로 묶어 출력해보도록 하세요. 기본 자료형을 만날 때 뿐만 아니라, 모듈에서 제공하고 있는 자료형이나 여러분이 작성한 자료형에서도 마찬가지입니다. 이를 통해 print로 출력되는 값, 검색할 키워드, 가진 속성 등을 확인할 수 있습니다. 이에 대한 내용은 각각의 자료형에서 상세히 다룹니다.

```python
print(a)
print(type(a))
print(dir(a))
```

# 7. 나아가기

앞서 언급한 매직 메서드를 조금 수정하면 아래와 같이 더하면 곱하기가 되게 하고, 곱하면 더하기가 되게 할 수도 있습니다. 여러분이 하시는 단순한 연산도 어딘가에 코드로 정의되어(짜여) 있다는 사실을 잊지 마세요.

```python-exec
class int(int):
    def __add__(self, a):
        return self * a

int('10') + int('10')
```
